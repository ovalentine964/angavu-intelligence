# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in the Angavu Intelligence website, please report it responsibly.

**DO NOT** open a public GitHub issue for security vulnerabilities.

Please email: **[INSERT SECURITY EMAIL]**

Include:
1. **Description** — What the vulnerability is
2. **Impact** — What could be affected
3. **Reproduction** — How to reproduce it

We will acknowledge within 48 hours and provide a fix timeline.

## Scope

This is a static website. Security concerns are limited to:

- Cross-site scripting (XSS)
- Content injection
- Service worker security
- Manifest file integrity

## Safe Harbor

We support responsible disclosure and will not take legal action against researchers who act in good faith.
